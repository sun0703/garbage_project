## 1. 架构设计概述

### 1.1 设计目标

本系统采用**零成本、全栈本地部署**的架构设计理念，以个人电脑作为服务器，通过内网穿透实现手机端访问。核心设计原则：

| 设计原则 | 说明 | 实现方式 |
|----------|------|----------|
| 零成本 | 无云服务器、无API费用、无域名备案 | 本地PC + Cpolar免费穿透 |
| 极简部署 | 一个Python文件 + 一个HTML文件即可运行 | FastAPI + Uvicorn |
| 高识别率 | PC端推理，算力充足，效果优于移动端 | YOLOv8n-cls ONNX |
| 双引擎交互 | 视觉识别 + 语音/文字搜索双通道 | ONNX Runtime + FuzzyWuzzy |

## 2. 核心组件角色分工

### 2.1 组件职责说明

| 组件 | 角色 | 技术选型 | 核心职责 |
|------|------|----------|----------|
| **FastAPI** | 大厨 | Python框架 | 编写业务逻辑、处理请求、调用模型、查询词库 |
| **Uvicorn** | 服务员 | ASGI服务器 | 接收HTTP请求、转发给FastAPI、返回响应结果 |
| **Cpolar** | 任意门 | 内网穿透工具 | 将本地端口暴露到公网，提供HTTPS安全链路 |
| **ONNX Runtime** | 推理引擎 | 微软推理库 | 加载ONNX格式模型并执行前向计算 |
| **YOLOv8n-cls** | 视觉大脑 | Ultralytics | 图像分类神经网络，输出物体类别概率 |
| **FuzzyWuzzy** | 容错大脑 | 字符串匹配库 | 模糊匹配用户输入与词库条目，处理拼写错误和同音字 |
| **waste.json** | 词库灵魂 | JSON映射表 | YOLO标签→中文垃圾分类→投放指南的完整映射 |

### 2.2 数据流转全景图

```
视觉识别流程：
手机拍照 → Base64编码 → HTTP POST → Uvicorn接收 → FastAPI路由
    ↓
Base64解码 → PIL.Image → 尺寸调整(224×224) → 归一化 → numpy数组
    ↓
ONNX Runtime推理 → 输出logits → Softmax → 取最大值 → 英文标签
    ↓
查 waste.json 映射表 → 中文分类名 + 垃圾桶颜色 + 投放指引
    ↓
JSON响应 → H5前端渲染 → 展示结果卡片


语音/文字搜索流程：
浏览器语音转文字 / 用户打字 → HTTP GET → Uvicorn接收 → FastAPI路由
    ↓
遍历 waste.json 所有条目 → FuzzyWuzzy 计算相似度分数
    ↓
取最高分匹配项 → 返回中文分类名 + 垃圾桶颜色 + 投放指引
    ↓
JSON响应 → H5前端渲染 → 展示搜索结果
```

## 6. 核心模块详细设计

### 6.1 视觉推理引擎 (VisionEngine)

#### 6.1.1 模块职责

加载YOLOv8n-cls ONNX模型，对上传的图片执行分类推理。

#### 6.1.2 类图设计

```
┌─────────────────────────────────────────┐
│           VisionEngine                 │
├─────────────────────────────────────────┤
│ - session: onnxruntime.InferenceSession │
│ - input_name: str                      │
│ - output_name: str                     │
│ - input_size: tuple                    │
│ - is_loaded: bool                      │
├─────────────────────────────────────────┤
│ + __init__(model_path: str)            │
│ + load_model(): None                   │
│ + predict(image_bytes): PredictResult  │
│ + preprocess(pil_image): np.ndarray    │
│ + postprocess(output): dict            │
│ + dispose(): None                      │
└─────────────────────────────────────────┘
```

#### 6.1.3 关键代码结构

```python
class VisionEngine:
    """视觉推理引擎：基于ONNX Runtime的图像分类"""

    def __init__(self, model_path: str):
        self.session = None
        self.input_name = ""
        self.output_name = ""
        self.input_size = (224, 224)
        self.is_loaded = False
        self._load_model(model_path)

    def _load_model(self, model_path: str):
        """加载ONNX模型"""
        import onnxruntime as ort
        self.session = ort.InferenceSession(model_path)
        self.input_name = self.session.get_inputs()[0].name
        self.output_name = self.session.get_outputs()[0].name
        self.is_loaded = True

    def predict(self, image: Image.Image) -> dict:
        """执行推理：输入PIL Image，输出分类结果"""
        input_tensor = self._preprocess(image)
        output = self.session.run(
            [self.output_name],
            {self.input_name: input_tensor}
        )
        return self._postprocess(output[0])

    def _preprocess(self, image: Image.Image) -> np.ndarray:
        """图像预处理：缩放→归一化→维度变换"""
        resized = image.resize(self.input_size)
        arr = np.array(resized).astype(np.float32) / 255.0
        mean = np.array([0.485, 0.456, 0.406])
        std = np.array([0.229, 0.224, 0.225])
        arr = (arr - mean) / std
        arr = arr.transpose(2, 0, 1)
        return np.expand_dims(arr, axis=0).astype(np.float32)

    def _postprocess(self, output: np.ndarray) -> dict:
        """后处理：Softmax→取最大值→返回结果"""
        exp_output = np.exp(output - np.max(output))
        probs = exp_output / exp_output.sum()
        top_idx = int(np.argmax(probs))
        confidence = float(probs[top_idx])
        return {"class_index": top_idx, "confidence": confidence}
```

### 6.2 模糊搜索引擎 (SearchEngine)

#### 6.2.1 模块职责

使用FuzzyWuzzy对用户输入进行模糊匹配，从词库中找到最相似的垃圾条目。

#### 6.2.2 类图设计

```
┌─────────────────────────────────────────┐
│           SearchEngine                  │
├─────────────────────────────────────────┤
│ - vocab: list[dict]                    │
│ - vocab_labels: list[str]              │
├─────────────────────────────────────────┤
│ + __init__(vocab_path: str)             │
│ + load_vocab(vocab_path: str): None    │
│ + search(query: str, top_k: int): list │
│ + _normalize(text: str): str           │
└─────────────────────────────────────────┘
```

#### 6.2.3 关键代码结构

```python
from fuzzywuzzy import process

class SearchEngine:
    """模糊搜索引擎：基于FuzzyWuzzy的容错匹配"""

    def __init__(self, vocab_path: str):
        self.vocab = []
        self.vocab_labels = []
        self._load_vocab(vocab_path)

    def _load_vocab(self, vocab_path: str):
        """加载垃圾分类词库JSON文件"""
        with open(vocab_path, 'r', encoding='utf-8') as f:
            data = json.load(f)
        self.vocab = data.get('items', [])
        self.vocab_labels = [item['label'] for item in self.vocab]

    def search(self, query: str, top_k: int = 3) -> list:
        """
        模糊搜索：返回top_k个最匹配的结果
        支持拼音近似、同音字、错别字等容错场景
        """
        results = process.extract(
            query,
            self.vocab_labels,
            limit=top_k
        )
        matched_results = []
        for label, score in results:
            item = next(
                (v for v in self.vocab if v['label'] == label), None
            )
            if item:
                matched_results.append({
                    **item,
                    'similarity_score': score
                })
        return matched_results
```

## 9. 性能指标

### 9.1 目标性能

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 模型加载时间 | < 3s | 首次请求耗时 |
| 单次推理时间 | < 100ms | CPU模式(YOLOv8n) |
| 图片上传处理 | < 500ms | 端到端(含网络) |
| 搜索响应时间 | < 50ms | FuzzyWuzzy匹配 |
| 并发处理能力 | ≥ 10 QPS | 本地测试 |

### 9.2 性能优化策略

| 优化点 | 方法 | 预期效果 |
|--------|------|----------|
| 模型预热 | 启动时执行一次空推理 | 消除首次推理延迟 |
| 图片压缩 | 前端压缩后再上传 | 减少传输时间 |
| 模型缓存 | ONNX Session复用 | 避免重复加载 |
| 词库预加载 | 启动时读入内存 | 搜索时直接内存查找 |

## 10. 安全考虑

### 10.1 安全措施

| 风险 | 应对措施 |
|------|----------|
| 图片大小攻击 | 限制上传图片≤5MB |
| 恶意请求 | 可添加速率限制（中间件） |
| XSS攻击 | 前端不执行外部脚本 |
| HTTPS强制 | Cpolar自动提供HTTPS证书 |

### 10.2 隐私保护

- 图片仅在本地处理，不上传至第三方服务器
- 不收集用户个人信息
- 识别完成后临时文件立即清理

## 11. 故障排查指南

### 11.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 手机无法打开摄像头 | 非HTTPS链接 | 必须用Cpolar的HTTPS地址 |
| 模型加载失败 | ONNX文件路径错误 | 检查models/目录下的文件 |
| 推理速度很慢 | 未使用GPU加速 | 安装onnxruntime-gpu版本 |
| Cpolar连接失败 | 网络问题或token过期 | 检查网络、重新绑定authtoken |
| 语音功能不可用 | 浏览器不支持 | Chrome/Edge/Safari均支持 |
