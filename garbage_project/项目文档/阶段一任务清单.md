## 1. 阶段目标

打造模块化 SPA 前端应用，达到 **「能拍照 → 能识别 → 能给出结果」** 的最小闭环用户体验。本阶段完成后，产品具备可演示的 Web 前端界面。

**核心交付物：**
- 模块化 SPA 前端（多页面路由、拍照组件、结果卡片）
- 完整的用户交互流程（上传 → 预览 → 识别 → 结果展示）
- 离线/弱网降级体验
- 识别历史本地存储

| 属性 | 内容 |
|------|------|
| **优先级** | **P0** |

| 错误场景 | 提示文案 |

| 500 服务端错误 | 「服务器繁忙，请稍后重试」 |
| 模型未加载 | 「AI 模型正在初始化，请稍候」 |

#### F-1.4.2 自动降级到特征分析模式

| 属性 | 内容 |
|------|------|
| **优先级** | **P0** |
| **描述** | 当 YOLO 模型不可用时（未加载/加载失败/超时），前端自动切换到基于图像特征分析的演示模式，仍能给出分类结果 |
| **实现位置** | 前端 `api.js` 中增加 fallback 逻辑 |
| **技术方案** | 主接口 `/api/predict` 失败且错误码为 `E2002`（模型未加载）时，自动调用特征分析备选接口 |
| **验收标准** | 模型卸载状态下的图片 → 仍能返回分类结果（置信度为模拟值，标注「演示模式」） |

**降级流程：**
```
POST /api/predict
    │
    ├── 成功 → 正常渲染结果卡片 ✅
    │
    └── 失败（E2002 模型不可用）
         │
         └── POST /api/debug/analyze（特征分析兜底）
              │
              ├── 成功 → 渲染结果卡片 + 标注「⚠️ 演示模式」⚠️
              └── 失败 → 显示「服务暂时不可用」错误页 ❌
```

## 4. SPA 重构工程任务 ⭐ 阶段一核心交付

当前 `index.html` 为单文件 Demo（所有 HTML/CSS/JS 内联），需拆分为模块化 SPA 结构。这是阶段一中**工作量最大、最关键的前端任务**，是后续所有阶段的前端基础。

### 4.1 当前状态 vs 目标状态

| 维度 | 当前状态 | 目标状态 |
|------|---------|---------|
| 文件结构 | 单个 `index.html`（~全量代码） | 多文件模块化（HTML + CSS + JS 分离） |
| 路由 | 无路由（单页静态） | Hash Router（6 个视图切换） |
| 组件 | 无组件化（内联 DOM 操作） | 独立 JS 组件（可复用） |
| 状态管理 | 全局变量 | 封装 Store 对象 |
| 构建工具 | 无（纯静态） | 无（Vanilla JS ES Module，零构建依赖） |

### 4.3 页面视图规划

| 视图 ID | 页面名称 | 路由 Hash | 核心功能 | 依赖模块 |
|---------|---------|-----------|---------|---------|
| `page-home` | 首页 | `#/home` 或 `#/` | 上传区域 + 搜索框 + 历史入口 | `home.js`, `camera-btn.js` |
| `page-preview` | 预览确认 | `#/preview` | 图片预览 + 重新选择 + 开始识别 | `camera.js` |

### 4.4 关键 JS 模块职责

#### `app.js` — 应用入口
- 初始化 Router、Store、API 客户端
- 绑定全局事件（hashchange、error、unhandledrejection）
- 启动首屏渲染

#### `api.js` — API 客户端
- 封装 `fetch()` 请求方法（统一 headers / 错误处理 / Token 注入）
- 提供 `predict()` / `batchPredict()` / `search()` 等业务方法
- 自定义 `ApiError` 错误类

#### `router.js` — Hash 路由器
- 注册路由表（path ↔ handler 映射）
- 监听 `hashchange` 事件切换视图
- 支持路由参数解析（如 `?q=keyword`）
- 默认路由（`#/` → `#/home`）重定向

#### `store.js` — 状态管理
- 全局状态对象（当前页面、用户信息、识别结果等）
- 简单发布订阅模式（事件驱动的状态更新）

#### `utils/image.js` — 图片处理
- `compress(file, maxSizeKB)` — Canvas 压缩至目标大小
- `toBase64(blob)` — Blob 转 DataURL
- `validate(file)` — 格式和大小校验

#### `utils/ui.js` — UI 工具

## 9. 风险与注意事项（前端）

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| Canvas 压缩在大图上耗时较长 | 用户等待感增加 | 显示压缩进度提示；设置最大分辨率上限 1280px |
| iOS Safari 不支持 `capture` 属性 | 无法直接调起相机 | iOS 会自动唤起原生相机选择器（相册+拍照），属于正常行为 |
| localStorage 有 5MB 限额 | 50 条缩略图可能超限 | 缩略图压缩到 ≤ 10KB；超出时自动清理最早记录 |
| ES Module 在旧浏览器不兼容 | 页面白屏 | 目标浏览器为 Chrome/Edge/Safari 最新版，暂不做 polyfill |
