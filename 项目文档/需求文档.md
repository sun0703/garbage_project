# 校园生活垃圾智能分类识别系统 - 需求文档

## 1. 项目概述

### 1.1 项目背景

随着全国垃圾分类政策的推进，校园场景下的垃圾智能分类识别需求日益增长。校园环境具有以下特点：
- **人流量集中**：课间、就餐后等时段产生大量垃圾
- **垃圾类型相对固定**：以餐余、纸张、塑料瓶、文具等为主
- **用户群体年轻化**：学生和教师对新技术的接受度高
- **硬件条件多样**：可能使用手机、平板或固定终端设备

### 1.2 项目目标

开发一套面向校园场景的轻量化生活垃圾智能分类识别系统，实现：
- 快速准确的垃圾图像识别
- 简单易用的用户交互界面
- 支持多种终端设备的部署能力
- 为校园垃圾分类教育提供辅助工具

### 1.3 项目范围

**本期范围（MVP）**：
- 移动端拍照识别功能
- 4-6类常见校园垃圾的分类识别
- 识别结果展示与投放指引
- 基础的用户反馈机制

**后续扩展**：
- 历史记录与数据统计
- AR投放指引功能
- 校园积分激励系统
- 多语言支持

## 3. 垃圾分类标准

### 3.1 校园常见垃圾类别

根据《上海市生活垃圾分类投放指南》并结合校园场景特点，定义以下分类标准：

| 类别 | 中文名称 | 英文名称 | 垃圾桶颜色 | 示例物品 |
|------|----------|----------|------------|----------|
| 1 | 厨余垃圾 | Kitchen Waste | 🟤 棕色 | 餐余食物、果皮、茶叶渣 |
| 2 | 可回收物 | Recyclable | 🔵 蓝色 | 塑料瓶、纸张、金属罐、玻璃瓶 |
| 3 | 其他垃圾 | Other Waste | 🟢 黑色/灰色 | 污染纸张、烟蒂、一次性餐具 |
| 4 | 有害垃圾 | Hazardous | 🔴 红色 | 废电池、过期药品、灯管 |

### 3.2 详细分类明细

#### 厨余垃圾（Kitchen Waste）
- 餐余食物（米饭、菜肴、骨头）
- 果皮果核（香蕉皮、苹果核）
- 茶叶渣、咖啡渣
- 蛋壳、虾壳
- 菜叶菜根

#### 可回收物（Recyclable）
- 塑料类：饮料瓶、洗发水瓶、塑料餐盒
- 纸张类：废纸、纸板、书本杂志
- 金属类：易拉罐、金属罐头、金属饮料瓶
- 玻璃类：玻璃瓶、玻璃杯
- 织物类：旧衣物（干净）、书包

#### 其他垃圾（Other Waste）
- 污损纸张（油污纸、纸巾）
- 烟蒂、烟盒
- 一次性餐具（筷子、餐盒）
- 尘土、灰土
- 破碎陶瓷
- 橡皮筋、扎带

#### 有害垃圾（Hazardous）
- 废电池（纽扣电池、充电电池）
- 废灯管（荧光灯、节能灯）
- 过期药品、化妆品
- 杀虫剂容器
- 温度计、水银血压计
- 油漆桶

## 4. 功能需求详细说明

### 4.1 图像采集模块

#### 4.1.1 拍照识别

**功能描述**：用户通过手机摄像头拍摄垃圾照片，系统自动进行分类识别

**业务流程**：
1. 用户点击"拍照识别"按钮
2. 打开系统相机
3. 用户拍摄照片
4. 用户确认或重新拍摄
5. 系统进行图像预处理
6. 调用模型进行推理
7. 显示识别结果

**输入**：
- 摄像头拍摄的图片（JPG格式）
- 或相册选择的图片

**输出**：
- 识别类别
- 置信度
- 投放指引

**性能要求**：
- 拍照到显示结果：≤500ms
- 图片尺寸：建议800x800像素

#### 4.1.2 实时识别

**功能描述**：开启摄像头后实时显示识别结果，无需拍照

**业务流程**：
1. 用户点击"实时识别"按钮
2. 打开摄像头预览
3. 系统持续进行目标检测和分类
4. 在预览画面上叠加显示识别结果
5. 用户点击屏幕拍照保存

**输入**：
- 摄像头实时流

**输出**：
- 实时分类标签
- 目标框标注

**性能要求**：
- 帧率：≥15fps
- 单帧识别延迟：≤100ms

### 4.2 模型推理模块

#### 4.2.1 图像预处理

**功能描述**：对输入图像进行标准化处理，适配模型输入要求

**处理流程**：
1. 读取图片数据
2. 调整图像尺寸（224x224 或 256x256）
3. 归一化处理（像素值 / 255.0）
4. 通道重排（HWC → CHW）
5. 均值减法（如 ImageNet 标准化）

**参数配置**：
```python
{
    "input_size": [224, 224],
    "mean": [0.485, 0.456, 0.406],
    "std": [0.229, 0.224, 0.225],
    "normalize": true
}
```

#### 4.2.2 模型推理

**功能描述**：使用训练好的模型对预处理后的图像进行分类

**模型选型要求**：
- 参数量：≤5MB
- 推理速度：≤50ms/帧（移动端）
- 支持INT8量化
- 兼容Android/iOS部署

**推荐模型**：
1. MobileNetV3-Small（优先）
2. EfficientNet-Lite0
3. ResNet18（量化版本）

#### 4.2.3 后处理

**功能描述**：对模型输出进行处理，生成最终分类结果

**处理流程**：
1. 读取模型输出 logits
2. 应用 Softmax 函数计算概率分布
3. 选取概率最大的类别
4. 应用置信度阈值过滤（默认0.5）
5. 映射类别ID到类别名称
6. 关联投放指引信息

### 4.3 结果展示模块

#### 4.3.1 识别结果卡片

**UI设计要求**：
- 清晰显示垃圾类别（中文大字）
- 显示英文名称（次要信息）
- 显示置信度百分比（带进度条）
- 显示投放指引（文字说明）
- 垃圾桶颜色标识

**显示示例**：
```
┌─────────────────────────┐
│      🔵 可回收物         │
│      Recyclable         │
│                         │
│  置信度：92%  ████████░░ │
│                         │
│  📌 请投入蓝色可回收物   │
│     垃圾桶               │
│                         │
│  [反馈错误]  [重新识别]   │
└─────────────────────────┘
```

#### 4.3.2 置信度展示

**展示规则**：
- ≥90%：显示为"很有把握"
- 70%-90%：显示为"比较确定"
- 50%-70%：显示为"建议核实"
- <50%：显示为"识别不确定"

**视觉设计**：
- 使用进度条或圆环图
- 颜色渐变：绿色（高）→ 黄色（中）→ 红色（低）

### 4.4 投放指引模块

#### 4.4.1 文字指引

**内容要求**：
- 垃圾类别名称
- 对应垃圾桶颜色
- 简单易懂的投放说明

**文案规范**：
```
厨余垃圾 → 请投放至棕色厨余垃圾桶
可回收物 → 请投放至蓝色可回收物垃圾桶
其他垃圾 → 请投放至灰色其他垃圾桶
有害垃圾 → 请投放至红色有害垃圾桶
```

#### 4.4.2 图示指引

**展示内容**：
- 垃圾桶图标
- 分类标识
- 简单动画（如有）

## 6. 约束条件

### 6.1 技术约束

- 模型必须在移动端可运行（Android/iOS）
- 模型文件大小不超过5MB
- 必须支持离线识别功能
- 推理延迟不超过500ms

### 6.2 资源约束

- 开发周期：MVP版本4-6周
- 开发团队：1-2人
- 测试设备：主流Android手机2-3台

### 6.3 法规约束

- 遵守《个人信息保护法》
- 图片数据本地处理，不上传服务器
- 用户反馈数据脱敏处理
